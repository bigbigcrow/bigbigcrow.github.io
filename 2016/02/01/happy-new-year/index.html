Hexo+Github Pages博客搭建历程

这是新的开始

Hexo + Github 搭建博客

A fast, simple & powerful blog framework, powered by Node.js.

准备工作：安装hexo之前，需要确保你已安装了node环境、github，这里就不多说了~

直接进入正题 - Hexo的安装

1、全局安装hexo

npm install -g hexo

2、创建hexo目录

cd ~
mkdir hexo

3、初始化hexo

cd hexo
hexo init

hexo文件夹下就会生成如下目录

├── _config.yml     // 网站的配置信息
├── package.json    // 应用程序信息，ejs、stylus、markdown已默认安转
├── scaffolds       // 模板文件
├── source          // 资源文件夹
|   ├── _posts      // 除_posts文件夹之外，开头命名为_(下划线)的文件、文件夹和隐藏的文件将会被忽略。
└── themes          // 主题文件

4、安装dependencies（依赖），生成node_modules目录

npm install

5、启动hexo服务（需先安转hexo-server模块）

hexo server

command + 单击（http://0.0.0.0:4000/）或者复制网址直接打开~

哇，成功了，可是为什么显示的和你的不一样！

莫急，这只是hexo默认的主题（themes/landscape），主题是可以自己配置的。

鉴于个人审美不同，认为默认主题也ok的，可忽略此步骤~

我的博客使用的theme是TKL（简洁大方，可以专心的看内容），下面就以此为例，来介绍下主题配置。

pwd
/Users/MLS/hexo
git clone https://github.com/SuperKieran/TKL.git
cp -r TKL themes
rm -rf TKL

如此一来，主题TKL就被复制进themes文件下了。主题下载完就该应用了

vi _config.yml

替换themes配置项值landscape为TKL，刷新下页面，新的theme已经生效了，接下来就是进入到themes/TKL/下的_config.yml替换下文本，logo，链接之类的

6、新建hexo文件（文件创建在source/_posts/下）

hexo new fileName

7、 配置~/hexo/_config.yml

配置~/hexo/_config.yml中deploy字段

vi ~/hexo/_config.yml

deploy:
type: git
repo: git@github.com:username/username.github.io.git
branch: master

cd ~/hexo
git init
git remote add origin git@github.com:username/repositoriesname
git add .
git commit -m message
git push -u origin master

8、生成静态文件（hexo/public）

hexo generate | g

9、

npm install –save hexo-deployer-git

10、部署（将网站部署到服务器上，_config.yml文件下配置的deploy字段）

hexo deploy

到此只能在本地4000端口上启动hexo服务，浏览博客，可别人也看不到，不爽中…
这回该轮到Github Pages出场了

在github中建立与你用户名对应的仓库，仓库名必须为username.github.io

cd ~
git clone https://github.com/username/username.github.io
cd username.github.io
cp -r ../hexo/public/ ~/username.github.io

可以直接访问username.github.io查看你的博客了

域名dns解析后，在username.github.io下创建CNAME文件，键入你的域名，推送到远程仓库即可~

输入你的域名，查看你的博客 @^-^@
